# 从零开始用 C++ 打造高性能 JSON 解析器（零）、前言

## 为什么我要写这个系列？

你好，C++ 开发者！

你是否已经熟悉了 C++ 的语法，却在面对“如何从零开始一个完整项目”时感到迷茫？你是否想深入理解一个工具的底层原理，而不仅仅是停留在调用 API 的层面？

本系列文章将带你完整走过一个 JSON 解析器从概念设计到代码实现的全过程。这不仅仅是一份施工指南，更是一次关于 **面向对象设计、内存管理、高性能优化及并发编程** 的实战演练。

所有代码已经放在 [github 仓库](https://github.com/pjh456/Let_us_build_cpp_json_parser)，感兴趣的读者可以自行查看！

## 成果预览：性能“爆杀” nlohmann/json？

在开始写代码前，我们先看一眼终点。我们将自己打造的解析器（PJH）与业界标杆 `nlohmann/json` 以及极致性能闻名的 `RapidJSON` 进行了对比，测试环境是处理一个 **40MB 的大型 JSON 文件**。

| Benchmark | Time (ns) | CPU (ns) | Iterations |
| :--- | :--- | :--- | :--- |
| **PJH (本项目)** | **242,114,575** | **191,406,250** | 4 |
| nlohmann/json | 840,837,650 | 578,125,000 | 2 |
| RapidJSON | 213,708,071 | 149,553,571 | 7 |

**结果分析：**
我们的解析器运行耗时不到 `nlohmann/json` 的 **1/3**，已经成功进入了由 `RapidJSON` 代表的业界第一梯队。

> 这份测试是很早以前写的，可能现版本测试时已经过时，且受到本地编译测试环境的影响，在此仅供参考。
